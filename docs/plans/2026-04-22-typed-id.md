# [TypedId] Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a `[TypedId]` source-generator attribute to `ZeroAlloc.ValueObjects` that emits strongly-typed identifier structs backed by `Guid` or `long`, with four generation strategies (ULID, UUID7, Snowflake, Sequential).

**Architecture:** New `TypedIdGenerator : IIncrementalGenerator` class added to `src/ZeroAlloc.ValueObjects.Generator` alongside the existing `ValueObjectGenerator`, separate parser/writer pipeline but shared utility code. Core runtime helpers (`UlidCore`, `Uuid7Core`, `SnowflakeCore`, `SequentialCore`) ship in `src/ZeroAlloc.ValueObjects` (netstandard2.0, BCL only). EF Core adapter ships as a new `ZeroAlloc.ValueObjects.EfCore` project in the same repo.

**Tech Stack:** .NET Standard 2.0 (attributes + generator), .NET 8/9/10 (runtime + tests), Roslyn 4.8, xUnit v2, Verify for snapshot tests, BenchmarkDotNet for allocation tests, SQLite in-memory for EF Core tests.

**Reference design:** [docs/plans/2026-04-22-typed-id-design.md](./2026-04-22-typed-id-design.md)

```csharp
namespace ZeroAlloc.ValueObjects.Tests;

public class UlidCoreTests
{
    [Fact]
    public void ToBase32_RoundTrips_ThroughTryFromBase32()
    {
        var g = UlidCore.NewGuid();
        var s = UlidCore.ToBase32(g);
        Assert.Equal(26, s.Length);
        Assert.True(UlidCore.TryFromBase32(s.AsSpan(), out var g2));
        Assert.Equal(g, g2);
    }

    [Fact]
    public void NewGuid_BurstOf1000_Monotonic()
    {
        Span<Guid> ids = stackalloc Guid[1000];
        for (int i = 0; i < 1000; i++) ids[i] = UlidCore.NewGuid();
        // ULIDs are lexicographically sortable — string-compare is sufficient
        for (int i = 1; i < 1000; i++)
            Assert.True(string.CompareOrdinal(UlidCore.ToBase32(ids[i - 1]), UlidCore.ToBase32(ids[i])) <= 0);
    }

    [Fact]
    public void TryFromBase32_RejectsInvalidLength() =>
        Assert.False(UlidCore.TryFromBase32("01ARZ3NDEKTSV4RRFFQ69G5FA".AsSpan(), out _));

    [Fact]
    public void TryFromBase32_RejectsInvalidCharacter() =>
        Assert.False(UlidCore.TryFromBase32("I1ARZ3NDEKTSV4RRFFQ69G5FAV".AsSpan(), out _));  // 'I' not in Crockford alphabet
}
```

**Step 2: Run to verify fail** — compile error.

**Step 3: Implement `UlidCore`**

Reference: ULID spec https://github.com/ulid/spec

Shape:
- `public static Guid NewGuid()` — 48-bit unix ms (big-endian, bytes 0-5) + 80-bit random (bytes 6-15). Uses `System.Security.Cryptography.RandomNumberGenerator.Fill(Span<byte>)` — zero-alloc, thread-safe. Convert to `Guid` via `new Guid(ReadOnlySpan<byte>, bigEndian: true)` (netstandard2.0 doesn't have that ctor — implement byte swap manually).
- `public static string ToBase32(Guid value)` — 16 bytes → 26-char Crockford base32 string. Alphabet: `0123456789ABCDEFGHJKMNPQRSTVWXYZ` (skip I, L, O, U). Stackalloc 26 chars.
- `public static bool TryFromBase32(ReadOnlySpan<char> s, out Guid value)` — parse 26 chars → 16 bytes → Guid. Reject unknown characters (return false, not throw).

The monotonicity guarantee requires a fall-back: if two ULIDs are generated in the same ms, increment the random portion. Simplest: cache last timestamp + last-random in a `[ThreadStatic]` or `Interlocked`-managed struct, and increment random on collision. For v1 the "Monotonic in burst" test above only requires non-decreasing order; true monotonic guarantee is a v2 concern. **Implement non-decreasing**: same-ms ULIDs get random portions generated independently but the timestamp prefix ensures non-decreasing across milliseconds. Document the v1 semantics in XML.

```csharp
// SnowflakeServiceCollectionExtensions.cs
using System;
using Microsoft.Extensions.DependencyInjection;
using Microsoft.Extensions.Hosting;

namespace ZeroAlloc.ValueObjects;

public static class SnowflakeServiceCollectionExtensions
{
    public static IServiceCollection AddSnowflakeWorkerId(this IServiceCollection services, int workerId)
        => AddSnowflakeWorkerId(services, _ => workerId);

    public static IServiceCollection AddSnowflakeWorkerId(this IServiceCollection services, string envVar, int fallback = 0)
        => AddSnowflakeWorkerId(services, _ =>
        {
            var raw = Environment.GetEnvironmentVariable(envVar);
            return int.TryParse(raw, out var v) ? v : fallback;
        });

    public static IServiceCollection AddSnowflakeWorkerId(this IServiceCollection services, Func<IServiceProvider, int> factory)
    {
        services.AddSingleton(factory);
        services.AddHostedService<SnowflakeWorkerIdStartup>();
        return services;
    }
}
```

```csharp
// SnowflakeWorkerIdStartup.cs
using System;
using System.Threading;
using System.Threading.Tasks;
using Microsoft.Extensions.Hosting;

namespace ZeroAlloc.ValueObjects;

internal sealed class SnowflakeWorkerIdStartup : IHostedService
{
    private readonly Func<IServiceProvider, int> _factory;
    private readonly IServiceProvider _sp;

    public SnowflakeWorkerIdStartup(Func<IServiceProvider, int> factory, IServiceProvider sp)
    {
        _factory = factory;
        _sp = sp;
    }

    public Task StartAsync(CancellationToken ct)
    {
        var id = _factory(_sp);
        if (id < 0 || id >= 1024)
            throw new TypedIdException($"Snowflake worker id {id} out of range [0, 1023].");
        TypedIdRuntime.SnowflakeProvider = new StaticProvider(id);
        return Task.CompletedTask;
    }

    public Task StopAsync(CancellationToken ct) => Task.CompletedTask;

    private sealed class StaticProvider : ISnowflakeWorkerIdProvider
    {
        public StaticProvider(int id) => WorkerId = id;
        public int WorkerId { get; }
    }
}
```

## Execution notes

- **Worktree:** create an isolated worktree on `feat/typed-id` via `superpowers:using-git-worktrees` before starting.
- **Analyzer settings:** `TreatWarningsAsErrors=true` with Meziantou + Roslynator + ErrorProne + ZeroAlloc + NetFabric.Hyperlinq stack. Every generator + runtime change must clear them without new suppressions unless the same pattern already exists.
- **Test framework:** xUnit v2, use `CancellationToken.None` not `TestContext.Current.CancellationToken`. No `ConfigureAwait(false)` inside test methods (xUnit1030).
- **Snapshot tests:** Verify library + existing setup. First run of a new snapshot test creates the `.received.cs` file; rename to `.verified.cs` after manual review.
- **TDD discipline:** every task: write test → fail → implement → pass → commit. Do not skip the fail step.
- **Netstandard2.0 gotchas:** no `Guid(ReadOnlySpan<byte>, bool)` ctor, no `Random.Shared`, no `CollectionsMarshal.AsSpan`. Implement byte-order helpers manually, use `RandomNumberGenerator.Fill(Span<byte>)`.
